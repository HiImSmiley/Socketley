Getting Started

Socketley is a high-performance Linux daemon and CLI tool that manages long-living network runtimes (servers, clients, proxies, caches) in a Docker-like style. Written in C++ with io_uring for async I/O and Lua scripting via LuaJIT.

Source code: github.com/HiImSmiley/socketley

Installation

From source (recommended)

git clone https://github.com/HiImSmiley/socketley.git
cd socketley
./bin/premake5 gmake2
cd make && make config=release_x64 -j$(nproc)

Binary output: bin/Release/socketley

System-wide install

sudo bash packaging/install.sh
# Or build a .deb:
bash packaging/build-deb.sh && sudo dpkg -i socketley_*.deb

Quick Start

# 1. Start the daemon
./bin/Release/socketley daemon &

# 2. Create and start a server
socketley create server myapp -p 9000 -s

# 3. Send a message (broadcasts to all clients)
socketley send myapp "Hello, world!"

# 4. Check status
socketley ls

# 5. Create a Redis-compatible cache
socketley create cache store -p 9001 --resp -s
redis-cli -p 9001 SET mykey myval

# 6. Interactive mode
socketley start myapp -i

Building from Source

./bin/premake5 gmake2
cd make
make config=release_x64 -j$(nproc)   # Release
make config=debug_x64 -j$(nproc)     # Debug

Requirements: Linux with io_uring (kernel 5.11+), GCC/Clang with C++20. All deps vendored in thirdparty/.

CLI Commands

All commands talk to the daemon via Unix socket. Start with socketley daemon. Most accept glob patterns (*, ?, []).

socketley create <type> <name> [flags]
Create a runtime. Types: server, client, proxy, cache. Add -s to start immediately.
socketley create server myapp -p 9000 -s
socketley create cache store -p 9001 --persistent /tmp/data.bin --maxmemory 256M -s
socketley create client bot -t 127.0.0.1:9000 --reconnect -s
socketley create proxy gw -p 8080 --backend 127.0.0.1:9000,127.0.0.1:9001 -s
socketley start <name|pattern>... [-i]
Start runtimes. -i = interactive (stdin/stdout). Ctrl+C to detach. Works on already-running runtimes.
socketley stop <name|pattern>...
Stop running runtimes. * stops all.
socketley remove <name|pattern>...
Stop and remove runtimes permanently.
socketley send <name> [message]
Send/broadcast. Supports stdin: echo "hi" | socketley send myapp
socketley stats <name|pattern>...
Show connections, messages, bytes in/out.
socketley reload <name|pattern>...
Restart runtimes (stop + start).
socketley reload-lua <name|pattern>...
Hot-reload Lua script without restarting.
socketley show <name|pattern>...
Print JSON config to stdout.
socketley edit <name> [flags]
With flags: apply directly. Without: opens $EDITOR. -r auto-reloads Lua.
socketley ls / ps / owner <name>
ls = all runtimes, ps = running only, owner = parent/children.
ID        NAME          TYPE    PORT    CONN  STATUS              CREATED
a1b2c3    api-server    server  9000    3     Up 2 hours          3 hours ago
d4e5f6    my-client     client  -       1     Up 45 minutes       1 hour ago
socketley --config <lua_file>
Load Lua orchestration config defining multiple runtimes.

Runtime Types

Server

TCP/UDP listener. Supports broadcast, per-client send, WebSocket auto-detection, master mode, cache access linking.

  • WebSocket: Auto-detected from HTTP upgrade. TCP and WS coexist on same port.
  • Master mode: One client broadcasts, others silenced or forwarded.
  • Cache access: --cache <name>, clients send cache <cmd>.

Client

TCP/UDP connector. Auto-reconnect with exponential backoff, message flow control.

Proxy

HTTP/TCP reverse proxy. Backends: IP:port or runtime names. Strategies: round-robin, random, Lua.

socketley create proxy gw -p 8080 --backend api1,api2 --strategy lua --config router.lua -s

Cache

In-memory store: strings, lists, sets, hashes, TTL, pub/sub, RESP2, LRU eviction, persistence, replication.

socketley create cache store -p 9000 --persistent /tmp/data.bin --maxmemory 256M --eviction allkeys-lru --resp -s

Cache Protocol

TCP newline-terminated commands (lowercase). Also supports RESP2 (auto-detected or --resp). Type enforcement: one type per key.

RESP2 Protocol

Force with --resp or auto-detected. Compatible with redis-cli and Redis client libraries.

socketley create cache myredis -p 6379 --resp -s
redis-cli -p 6379 SET mykey myval

# Replication
socketley create cache leader -p 9000 -s
socketley create cache follower -p 9001 --replicate 127.0.0.1:9000 -s

Lua Scripting

Socketley embeds LuaJIT via sol2. Scripts attach to runtimes for lifecycle callbacks and action methods.

Attaching Scripts

# Per-runtime
socketley create server myserver -p 9000 --config handler.lua -s

# Orchestration (defines runtimes table)
socketley --config setup.lua

How Bindings Work

Binding order: register_bindings() runs before script load. self, socketley, and type alias are available at top level.
  1. Fresh LuaJIT state with base, string, table, math, os
  2. Registers socketley namespace
  3. Registers self table with properties + action methods
  4. Registers type alias (server/client/cache/proxy) pointing to self
  5. Executes script, captures callbacks

Lua Callbacks

Define global functions for lifecycle events. All optional.

self.state is updated before each callback.
on_start()all types
Fires when runtime enters running state.
function on_start()
    socketley.log(self.name .. " started on port " .. tostring(self.port))
end
on_stop()all types
Fires when runtime enters stopped state.
on_message(msg)serverclientcache
Fires on received message. Trailing newline stripped.
function on_message(msg)
    self.broadcast("[echo] " .. msg)
end
on_client_message(client_id, msg)server
Fires with sender ID. Also fires on_message.
function on_client_message(client_id, msg)
    if msg == "whoami" then
        self.send(client_id, "You are #" .. tostring(client_id))
    end
end
on_connect(client_id)servercache
Fires when a client connects.
on_disconnect(client_id)servercache
Fires when a client disconnects.
on_send(msg)serverclient
Fires after a message is sent/broadcast.
on_route(method, path)proxy
Fires per HTTP request when strategy is "lua".
Returns: number (0-based backend index) or nil
function on_route(method, path)
    if path:match("^/api/") then return 0 end
    return nil
end
on_master_auth(client_id, password)server
Fires on master claim (master <password>).
Returns: boolean

Lua Server API

self.broadcast(msg)server
Send to all connected clients. Newline appended. WS clients get framed text.
self.send(client_id, msg)server
Send to a specific client by fd.
self.connections() → numberserver
Current client count.

Master Mode

With --mode master, one client broadcasts. Auth via --master-pw or on_master_auth.

Lua Client API

self.send(msg)client
Send to connected server.
self.connections() → numberclient
0 = disconnected, 1 = connected.
Auto-reconnect: --reconnect [max] enables exponential backoff. on_start fires again on reconnect.

Lua Cache API

Direct in-memory access from Lua, no TCP round-trip.

Type enforcement: One type per key. Use self.del(key) to clear.

Strings

self.set(key, value) → booleancache
self.get(key) → stringcache
self.del(key) → booleancache

Lists

self.lpush / self.rpush(key, value) → booleancache
self.lpop / self.rpop(key) → stringcache
self.llen(key) → numbercache

Sets

self.sadd(key, member) → numbercache
self.srem(key, member) → booleancache
self.sismember(key, member) → booleancache
self.scard(key) → numbercache

Hashes

self.hset(key, field, value) → booleancache
self.hget(key, field) → stringcache
self.hdel(key, field) → booleancache
self.hlen(key) → numbercache

TTL / Expiry

self.expire(key, seconds) → booleancache
self.ttl(key) → numbercache
-1 = no TTL, -2 = not found.
self.persist(key) → booleancache

Pub/Sub

self.publish(channel, message) → numbercache
Publish to channel. Returns subscriber count.

Lua Proxy API

self.connections() → numberproxy

Lua Routing

Set strategy to "lua", backends are 0-indexed. Use on_route callback.

Runtime Management API

socketley.create(type, name, config?) → booleanall
Config: port, config/lua, target, mode, on_parent_stop, autostart.
socketley.start(name) → booleanall
socketley.stop(name) → booleanall
socketley.remove(name) → booleanall
socketley.send(name, msg) → booleanall
socketley.list() → tableall
Array of all runtime names.
socketley.get(name) → table or nilall
Returns {name, type, state, port, connections, owner}.

Ownership

Lua-created runtimes are owned by caller. on_parent_stop = "remove" for auto-cleanup.

socketley.create("server", "worker", {
    port = 0, config = "worker.lua",
    on_parent_stop = "remove", autostart = true
})

Client Routing

Forward clients to child sub-servers. For game lobbies, chat rooms, microservices.

Routed clients' messages go to sub-server's on_client_message. Sub-servers respond via owner_send/owner_broadcast.
self.route(client_id, target) → booleanserver
self.unroute(client_id) → booleanserver
self.get_route(client_id) → string or nilserver
self.owner_send(client_id, msg) → booleanserver
From sub-server, send to parent's client.
self.owner_broadcast(msg) → booleanserver
From sub-server, broadcast to parent's clients.

Port-0 servers have no listener — only receive via client routing.

Examples

Echo Server

function on_message(msg)
    self.broadcast(msg)
end
socketley create server echo -p 9000 --config echo.lua -s

Chat Server

local clients = {}

function on_connect(client_id)
    clients[client_id] = "user" .. client_id
    self.send(client_id, "Welcome! You are " .. clients[client_id])
    self.broadcast(clients[client_id] .. " joined")
end

function on_disconnect(client_id)
    local name = clients[client_id] or "unknown"
    clients[client_id] = nil
    self.broadcast(name .. " left")
end

function on_client_message(client_id, msg)
    if msg:match("^/nick ") then
        local old = clients[client_id]
        clients[client_id] = msg:match("^/nick (%S+)")
        self.broadcast(old .. " is now " .. clients[client_id])
        return
    end
    self.broadcast("[" .. clients[client_id] .. "] " .. msg)
end

Game Lobby

function on_start()
    socketley.create("server", "chess-rapid", {
        port = 0, config = "rapid-handler.lua",
        on_parent_stop = "remove", autostart = true
    })
end

function on_client_message(client_id, msg)
    if msg:match("^join ") then
        self.route(client_id, "chess-" .. msg:match("^join (%S+)"))
    end
end

Cache-Backed API

function on_start()
    self.set("config:version", "1.0.0")
    self.hset("user:1", "name", "Alice")
    socketley.log("Cache seeded")
end

Master Broadcast

local tokens = { ["admin-secret"] = true }
function on_master_auth(client_id, password)
    return tokens[password] == true
end
socketley create server ctrl -p 9000 --mode master --master-forward --config master.lua -s

Proxy with Lua Routing

runtimes = {
    { type = "server", name = "static", port = 9001, autostart = true },
    { type = "server", name = "api", port = 9002, autostart = true },
    { type = "proxy", name = "gw", port = 8080,
      strategy = "lua", backends = { "127.0.0.1:9001", "127.0.0.1:9002" },
      autostart = true }
}
function on_route(method, path)
    if path:match("^/static/") then return 0 end
    return 1
end
socketley --config smart-router.lua
